About this interface

Web frontend for the ECC toolkit at github.com/VivekHalder/Crytography. The browser is doing none of the math. Each request shells out to sage KeyGeneration.py, sage Encrypt.py, or sage Decrypt.py on the server, with arguments built from the form. Output JSON is read back from disk and returned.

The protocol

ElGamal-style ECC over a Weierstrass curve E defined over GF(pn). Encryption picks a random ephemeral k, sends C₁ = kG and C₂ = M + k·PubKey. Decryption recovers M = C₂ − sk·C₁. ASCII mode (mode 1) packs UTF-8 bytes into the x-coordinate via lift_x with a one-byte trailing nonce so a valid y can be found; mode 2 takes the points directly.

Curves available

Mode 0/1/3 build curves dynamically. Mode 2 ships with secp256k1, P-256, and Curve25519 (in its underlying Weierstrass / Montgomery form).

Caveats

This is teaching/research code, not a hardened library. Don't encrypt secrets you actually care about with it. For production, use libsodium or the platform's vetted ECDH/HPKE.
